# 🚀 Guía de Instalación y Desarrollo\n\nInstrucciones completas para instalar, ejecutar y desarrollar Tienda Zerelle.\n\n## 📋 Tabla de Contenidos\n\n- [Requisitos Previos](#requisitos-previos)\n- [Instalación Inicial](#instalación-inicial)\n- [Configuración Firebase](#configuración-firebase)\n- [Ejecutar la Aplicación](#ejecutar-la-aplicación)\n- [Desarrollo](#desarrollo)\n- [Troubleshooting](#troubleshooting)\n- [Contribución](#contribución)\n\n---\n\n## ✅ Requisitos Previos\n\n### Software Requerido\n\n| Requisito | Versión | Descargar |\n|-----------|---------|----------|\n| Node.js | v16+ | [nodejs.org](https://nodejs.org) |\n| npm | v8+ | Incluido con Node.js |\n| Expo CLI | Última | `npm install -g expo-cli` |\n| Git | Cualquiera | [git-scm.com](https://git-scm.com) |\n\n### Para Android\n- Android Studio o Android SDK\n- JDK 11+\n- Android SDK nivel 26+\n\n### Para iOS (macOS)\n- Xcode 13+\n- CocoaPods\n- Simulador iOS o dispositivo iPhone\n\n### Para Web\n- No requiere configuración adicional\n\n---\n\n## 🔧 Instalación Inicial\n\n### Paso 1: Clonar el repositorio\n\n```bash\ngit clone https://github.com/tu-usuario/Proyecto-Zerelle.git\ncd Proyecto-Zerelle\n```\n\n### Paso 2: Instalar dependencias\n\n```bash\nnpm install\n# o con yarn\nyarn install\n```\n\n### Paso 3: Crear archivo `.env`\n\nEn la raíz del proyecto, crear ``.env`` con credenciales de Firebase:\n\n```bash\nEXPO_PUBLIC_FIREBASE_API_KEY=tu_api_key_aqui\nEXPO_PUBLIC_FIREBASE_AUTH_DOMAIN=tu_auth_domain.firebaseapp.com\nEXPO_PUBLIC_FIREBASE_PROJECT_ID=tu_project_id\nEXPO_PUBLIC_FIREBASE_STORAGE_BUCKET=tu_storage_bucket.appspot.com\nEXPO_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=tu_sender_id\nEXPO_PUBLIC_FIREBASE_APP_ID=tu_app_id\n```\n\n> ⚠️ **Importante:** Nunca hacer commit de `.env` a git. Agregar a `.gitignore`\n\n### Paso 4: Verificar instalación\n\n```bash\nnpm start\n```\n\nDebe mostrarse el menú de Expo.\n\n---\n\n## 🔥 Configuración Firebase\n\n### Crear Proyecto Firebase\n\n1. Esperar a que se cree\n\n### Habilitar Autenticación\n\n1. En el menú lateral: **Authentication**\n2. Click en \"Comenzar\"\n3. Seleccionar **Email/Password**\n4. Habilitar\n\n### Crear Firestore Database\n\n1. Menú lateral: **Firestore Database**\n2. Click en \"Crear base de datos\"\n3. Seleccionar región (ej: Sudamérica - Sao Paulo)\n4. Seleccionar modo: **Modo de desarrollo** (para desarrollo local)\n5. Click en \"Crear\"\n\n### Copiar Credenciales\n\n1. Menú lateral: **Project Settings** (engranaje)\n2. Pestaña: **General**\n3. Descargar config (SDKs > Aplicación web)\n4. Copiar las variables a `.env`\n\n### Crear Colecciones en Firestore\n\n#### 1. Título a mostrar\n * @param onPress - Callback al presionar\n */\nexport function MiComponente({ titulo, onPress }: Props) {\n  return (\n    <TouchableOpacity onPress={onPress}>\n      <Text>{titulo}</Text>\n    </TouchableOpacity>\n  );\n}\n```\n\n### Crear un Nuevo Hook\n\n**Archivo:** `hooks/useMiHook.ts`\n\n```typescript\nimport { useState, useEffect } from 'react';\n\n/**\n * Hook para hacer algo especial\n * \n * @returns Estado y funciones\n */\nexport function useMiHook() {\n  const [estado, setEstado] = useState(false);\n\n  useEffect(() => {\n    // Lógica\n  }, []);\n\n  return { estado, setEstado };\n}\n```\n\n**Actualizar:** `hooks/index.ts`\n\n```typescript\nexport { useMiHook } from './useMiHook';\n```\n\n### TypeScript\n\nTodos los archivos deben ser `.ts` o `.tsx` con tipos completos.\n\n```typescript\n// ❌ Evitar\nfunction miFunc(data) {\n  return data;\n}\n\n// ✅ Preferir\nfunction miFunc(data: Product): Product {\n  return data;\n}\n```\n\n### Estilo de Código\n\n**Convenciones:**\n- Componentes: PascalCase (`ProductCard.tsx`)\n- Hooks: camelCase con prefijo `use` (`useProducts.ts`)\n- Tipos: PascalCase (`Product`, `CartItem`)\n- Constantes: UPPER_SNAKE_CASE (`MAX_ITEMS = 100`)\n- Variables: camelCase (`userName`)\n\n### Hot Reload\n\nLos cambios se recargan automáticamente. Si no funciona:\n\n```bash\n# En el terminal de Expo\nr  # Recargar\ncc # Limpiar caché\n```\n\n---\n\n## 🧪 Testing\n\n### Ejecutar Tests\n\n*Actualmente sin configuración de tests*\n\nPara implementar:\n\n```bash\nnpm install --save-dev jest @testing-library/react-native\n```\n\n### Ejemplo de Test\n\n```typescript\n// __tests__/hooks/useProducts.test.ts\nimport { renderHook } from '@testing-library/react-native';\nimport { useProducts } from '@/hooks';\n\ntest('useProducts carga productos', async () => {\n  const { result } = renderHook(() => useProducts());\n  \n  // Esperar a que cargue\n  await waitFor(() => {\n    expect(result.current.loading).toBe(false);\n  });\n  \n  // Verificar datos\n  expect(result.current.products.length).toBeGreaterThan(0);\n});\n```\n\n---\n\n## 🐛 Troubleshooting\n\n### Error: \"Cannot find module '@/hooks'\"\n\n**Causa:** Los alias de path no están configurados\n\n**Solución:**\n1. Verificar `tsconfig.json`:\n   ```json\n   {\n     \"compilerOptions\": {\n       \"baseUrl\": \".\",\n       \"paths\": {\n         \"@/*\": [\"./*\"]\n       }\n     }\n   }\n   ```\n2. Reiniciar Expo: `npm start` luego `cc` (limpiar caché)\n\n---\n\n### Error: \"Firebase initialization failed\"\n\n**Causa:** Variables de entorno incorrectas o faltantes\n\n**Solución:**\n1. Verificar que `.env` existe\n2. Copiar credenciales correctas de Firebase Console\n3. Usar prefijo `EXPO_PUBLIC_` en nombres de variables\n4. Reiniciar Expo: `npm start`\n\n---\n\n### Error: \"Camera permission denied\"\n\n**Causa:** Permisos no solicitados al usuario\n\n**Solución:**\n1. En Android: Se solicita en tiempo de ejecución (automático)\n2. En iOS: Agregar a `app.json`:\n   ```json\n   {\n     \"ios\": {\n       \"infoPlist\": {\n         \"NSCameraUsageDescription\": \"Permitir acceso a la cámara\",\n         \"NSPhotoLibraryUsageDescription\": \"Permitir acceso a fotos\"\n       }\n     }\n   }\n   ```\n\n---\n\n### Error: \"Module not found\" después de instalar paquete\n\n**Solución:**\n```bash\nnpm install\nnpm start\n# Presionar cc (limpiar caché)\n```\n\n---\n\n### La aplicación es lenta o se cuelga\n\n**Causas comunes:**\n1. Muchas imágenes sin lazy loading\n2. Componentes re-renderizándose innecesariamente\n3. Loops infinitos en effects\n\n**Soluciones:**\n1. Usar `Image` de Expo con carga lazy\n2. Usar `useMemo` y `useCallback`\n3. Verificar dependencias en `useEffect`\n\n---\n\n### \"Something went wrong while logging in\"\n\n**Causa:** Credenciales incorrectas o usuario no existe\n\n**Solución:**\n1. Verificar email y contraseña\n2. Ir a Firebase Console > Authentication\n3. Crear usuario manual si es necesario\n\n---\n\n## 📱 Comandos Útiles\n\n```bash\n# Iniciar desarrollo\nnpm start\n\n# Android\nnpm run android\n\n# iOS\nnpm run ios\n\n# Web\nnpm run web\n\n# Limpiar caché\nnpm start\n# Luego presionar 'cc'\n\n# Instalar dependencias nuevas\nnpm install nombre-del-paquete\n\n# Desinstalar paquete\nnpm uninstall nombre-del-paquete\n\n# Ver versiones instaladas\nnpm list\n```\n\n---\n\n## 🔐 Seguridad en Desarrollo\n\n### Nunca Commitear\n```\n.env\n.env.local\nnode_modules/\n.expo/\n```\n\n### Verificar .gitignore\n```bash\ncat .gitignore | grep .env\n```\n\n### Variables Sensibles\n- Mantener en `.env` local\n- Nunca en comentarios de código\n- Usar variables de entorno en build\n\n---\n\n## 🤝 Contribución\n\n### Flujo de Trabajo\n\n1. **Crear rama**\n   ```bash\n   git checkout -b feature/nueva-feature\n   ```\n\n2. **Hacer cambios**\n   ```bash\n   # Editar archivos\n   git add .\n   git commit -m \"Agregar nueva feature\"\n   ```\n\n3. **Enviar cambios**\n   ```bash\n   git push origin feature/nueva-feature\n   ```\n\n4. **Pull Request**\n   - Ir al repositorio en GitHub\n   - Click en \"New Pull Request\"\n   - Completar descripción\n   - Esperar review\n\n### Convenciones de Commits\n\n```\nfeat: Agregar nueva feature\nfix: Corregir bug\nrefactor: Reorganizar código\ndocs: Actualizar documentación\nstyle: Cambios de formato\ntest: Agregar tests\n```\n\n---\n\n## 📚 Recursos Adicionales\n\n- [Documentación de Hooks](./HOOKS.md)\n- [Arquitectura del Proyecto](./ARQUITECTURA.md)\n- [Expo Documentation](https://docs.expo.dev)\n- [React Native Docs](https://reactnative.dev)\n- [Firebase Documentation](https://firebase.google.com/docs)\n- [Redux Toolkit](https://redux-toolkit.js.org)\n- [TypeScript Handbook](https://www.typescriptlang.org/docs)\n\n---\n\n## 💬 Soporte\n\nPara problemas o dudas:\n\n1.
